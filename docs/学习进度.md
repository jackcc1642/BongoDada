# BongoDada 学习进度

## 项目定位

BongoDada 是一个 Godot 4.5 桌面小宠物练习项目，对标 bongo cat。

当前阶段目标是完成 v0.1：透明置顶窗口、键盘敲击动画、计数、皮肤解锁、存档、极简菜单。先追求完整闭环，不做复杂养成、小屋、经营、剧情。

## 协作节奏

默认使用教练模式：

- 先解释现有代码和概念。
- 每次只给一个小任务。
- 用户亲手修改，AI 负责检查 diff、解释报错、指出最小修复。
- 不要直接替用户写完整大功能，除非用户明确说切到副驾模式。
- 每天结束可以让 AI 检查改动并上传 GitHub。

常用提示：

```text
请用教练模式。
不要直接替我写完整功能。
先解释我需要理解的代码，再给我一个小步骤任务。
我写完后你帮我检查。
```

## 当前已完成

- 按键检测：`_input(event)` 监听键盘输入，排除长按 echo。
- 敲击动画：`hands1` / `hands2` 两张图来回切换，停止后回到 idle。
- 计数：`Config.total_hits` 记录敲击次数，底部计数按钮实时显示。
- 皮肤配置：`Config.skins` 管理皮肤资源路径和位置/缩放/旋转参数。
- 解锁配置：`Config.unlock_config` 用敲击次数作为门槛。
- 解锁领取：达到门槛弹气泡，点击后加入已领取和已解锁列表。
- 领取后切换：点击奖励气泡后会切到刚领取的皮肤。
- 存档：保存 `total_hits`、`unlocked_skins`、`claimed_rewards`、`current_skin_id`。
- 读档：启动时读取存档，再刷新计数和当前皮肤。
- 未领取补弹：已经达到门槛但未领取的奖励，重开游戏后仍会弹出。
- 简单菜单：菜单按钮可打开/关闭面板。
- 菜单计数：菜单显示敲击次数，菜单打开时敲击会实时刷新。
- 菜单切换皮肤：点击按钮会在已解锁皮肤中循环切换，并保存当前皮肤。
- 菜单当前皮肤：菜单会显示当前皮肤中文名，切换皮肤后同步刷新。
- 重置存档：菜单提供重置按钮，确认后恢复默认数据、默认皮肤并保存。
- 重置确认：重置前会弹出确认框，取消不会清空数据。
- 菜单位置：菜单面板已调整为更稳定的小面板位置和最小宽度。
- 敲击形变：小马缩放中心改用 `TextureRect` 实际尺寸，修复敲击时整体下跳的问题。

## 已学概念

- 玩家数据和配置数据的区别：
  - 玩家数据需要保存，例如敲击次数、已解锁皮肤、已领取奖励、当前皮肤。
  - 配置数据不需要保存，例如解锁表、皮肤资源表。
- `find()`：
  - 在数组中查找某个值的位置。
  - 找到返回索引，例如 `0`、`1`、`2`。
  - 找不到返回 `-1`。
- `visible = not visible`：
  - 用来切换 UI 显示/隐藏。
- UI 刷新原则：
  - 数据变化时刷新相关 UI。
  - UI 打开时刷新一次当前状态。
  - 不要为了简单文字每帧刷新。
- 确认框流程：
  - 危险操作先弹 `ConfirmationDialog`。
  - 普通按钮只负责打开确认框。
  - 真正的清空逻辑放在确认框的 `confirmed` 信号里。
- UI 缩放中心：
  - `TextureRect` 的实际显示尺寸不一定等于图片原始尺寸。
  - 做敲击压缩这类 `scale` 动画时，优先用节点 `size` 设置 `pivot_offset`，避免视觉位置跳动。
- Git 节奏：
  - 每次完成一个小闭环后提交。
  - commit 信息尽量使用中文，口语化描述今天做了什么。
  - 每次让 AI 帮忙提交/上传前，AI 要先自动更新本文件的学习进度，再和代码改动一起提交；用户不需要单独提醒。
